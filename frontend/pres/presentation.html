Целевая аудитория и потребности

Студенты

  • Мгновенный доступ к расписанию, проектам, библиотеке.
  • Уведомления по дедлайнам и изменениям.

Абитуриенты

  • Навигация по программам и условиям поступления.
  • Помощь с документами и открытыми днями.

Сотрудники

  • HR/деканат: разгрузка от повторяющихся вопросов.
  • Единые процессы гостевых пропусков, отпусков, командировок.

Проблемы аудитории

  • Сложный доступ к функциям: у каждого сервиса свой интерфейс, нужен универсальный вход.
  • Личный кабинет вуза перегружен ссылками и инструкциями — пользователи теряются.
  • Все процессы выведены на сайт: много форм, бумажных заявок, очередей.
  • Нет универсального решения, которое «понимало бы» контекст пользователя и вело по сценариям.

Технические требования

Универсальность

Возможность подключить любой вуз: если есть API — используем, если нет — строим адаптер поверх выгрузок.

Масштабируемость

Kubernetes + autoscaling. Новые сценарии подключаются как микросервисы, не ломая общую схему.

Интерфейс

Строгая привязка к MAX WebApp UI: те же карточки, типографика, нижнее меню, skeleton-стейты.

Безопасность

SSO MAX ID, разграничение ролей, шифрование диалогов, журналирование, готовность к аудитам вузов.

Ключевые фичи · абитуриент

1. Подбор программ

Бот задаёт вопросы, подбирает направления, открывает карточки ProgramsPage.

2. Поступление

Ведёт по сценариям Admission Inquiry: собирает данные, отправляет заявку через API.

3. Open Day

Регистрирует на дни открытых дверей, выдаёт QR, напоминает о визите.

Ключевые фичи · студент

1. Персональное расписание

«Покажи пары завтра» — бот возвращает табы SchedulePage, можно пролистать недели.

2. Проекты и таски

Интеграция с ProjectsPage и TaskTracker: подача заявки, проверка статусов, уведомления.

3. Сервисы кампуса

Деканат, библиотека, общежитие — бот заполняет формы, отслеживает ответ.

Ключевые фичи · сотрудник

1. Командировки

Бот собирает цель поездки, даты, бюджет. Данные уходят в StaffTravelPage API.

2. Отпуска и гостевые

Запрашивает необходимые поля, подставляет контакты замены, фиксирует статусы.

3. HR справки

Коннектор к HR сервиcам: бот подбирает тип справки, отправляет заявку, сообщает о готовности.

Архитектура и масштабирование

Единая логика

Клиентское приложение не меняется: это тот же MAX WebApp. Чат-бот добавляет слой диалогов поверх UI и вызывает существующие эндпоинты.

Гибрид API

Если у вуза есть API — подключаем напрямую. Если нет — строим адаптер (ETL + наш REST), конвертируем данные к единому контракту.

Уровень карты

API Gateway + Orchestrator. Интенты → сценарии → вызов нужных сервисов.

Adapter Layer

Набор микросервисов для проектов, расписания, библиотек, HR. Каждый может подменить источник данных.

Масштабирование

Добавление новых вузов — лишь настройка маппингов. Планы: plug-in под корпоративные порталы, аналитика usage.
